OpenShiftは、プラットフォームと対話するための2つの主要なインターフェースを提供します。ビジュアル管理のためのWebベースのコンソールと、スクリプティングと高度な操作のためのコマンドラインインターフェースです。
このセクションでは、両方のツールに効果的にアクセスして使用する方法を学びます

[#the_web_console]
== Webコンソール

OpenShiftコンソールは、プロジェクト、アプリケーション、およびクラスターリソースを管理するための直感的なグラフィicalインターフェースを提供します。
最新のWebブラウザからアクセスでき、一般的な管理タスクと開発タスクを簡素化するように設計されています。

=== OpenShift Webコンソールへのアクセス

[.bordershadow]
image::openshift-console-login.png[width=500,link="self",window=_blank]

OpenShiftコンソールにアクセスするには、ブラウザで次のURLに移動し、提供された認証情報を使用してログインします：

*コンソールURL:* link:{ocp4_starter_console_url}[{ocp4_starter_console_url},window='_blank']

*ユーザー名:* `{user}`

*パスワード:* `{ocp4_starter_password}`

ログインすると、メインダッシュボードが表示され、プロジェクトを表示したり、プラットフォームの機能をナビゲートしたりできます。

[.bordershadow]
image::openshift-console-dashboard.png[link="self",window=_blank]

[#command_line_interface]
== コマンドラインインターフェース

Webコンソールはビジュアル管理に優れていますが、OpenShift CLI（`oc`）は強力な自動化機能と高度な機能へのアクセスを提供します。
CLIはGoで書かれた単一の実行可能ファイルであり、Microsoft Windows、macOS、およびLinuxで利用できます。

=== CLIインストールの確認

CLIはワークショップ環境にプリインストールされています。
利用可能であることを確認し、バージョンをチェックしてください：

[source,role="copypaste",subs="attributes"]
----
oc version
----

以下のような出力が表示されるはずです：

[.console-output]
[source,subs="attributes"]
----
Client Version: {ocp4_starter_cli_version}
Kustomize Version: v5.5.0
Kubernetes Version: v1.32.6
----

=== CLIでのログイン

コマンドラインを使用してOpenShiftクラスターで認証します：

[source,role="copypaste",subs="attributes"]
----
{ocp4_starter_login}
----

このコマンドは、ユーザー名とパスワードを使用してクラスターに対して認証します。
NOTE: 「certificate signed by unknown authority（不明な認証局によって署名された証明書）」というメッセージが表示された場合、このワークショップ環境では無視してかまいません。
これは、クラスターのAPI証明書が自己署名であることを示しており、開発環境やテスト環境では一般的です。

[.bordershadow]

TIP: または、トークンを使用して認証することもできます。トークンを取得するには、図のようにOpenShift Webコンソールの **Copy login command（ログインコマンドのコピー）** オプションを使用します。

[.bordershadow]
image::openshift-console-login-command.png[link="self",window=_blank]

=== アクセスの確認

適切に認証され、割り当てられたリソースにアクセスできることを確認しましょう。
まず、ログイン認証情報を確認します：

[source,role="copypa
ste",subs="attributes"]
----
oc whoami
----

ユーザー名が表示されるはずです：

[.console-output]
[source,subs="attributes"]
----
{ocp4_starter_username}
----

次に、割り当てられているプロジェクトを表示できることを確認します：

[source,role="copypaste",subs="attributes"]
----
oc projects
----

次の例のような出力が表示されるはずです：

[.console-output]
[source,subs="attributes"]
----
You have one project on this server: "Workspace ({ocp4_starter_project})".
Using project "{ocp4_starter_project}" on server "{ocp4_starter_api_url}".
----

素晴らしい！これで、Webコンソールとコマンドラインインターフェースの両方からOpenShiftにアクセスできるようになりました。
