== OpenShiftとは？

link:https://www.redhat.com/en/technologies/cloud-computing/openshift[OpenShift,window='_blank'] は、コンテナベースのアプリケーションプラットフォームであり、開発チームがオンプレミス、パブリッククラウド、ハイブリッド環境など、あらゆるインフラストラクチャ全体でアプリケーションをデプロイ、管理、スケーリングできるようにします。

== コンテナとは？

link:https://www.redhat.com/en/topics/containers/whats-a-linux-container-vb[コンテナ,window='_blank']は、アプリケーションと（訳注：それに）必要なすべての依存関係を一緒にパッケージ化する標準的な方法です。
このアプローチにより、デプロイが簡素化され、デリバリーが加速されます。仮想マシンとは異なり、コンテナはオペレーティングシステム全体を含みません。
アプリケーション、ランタイム、ライブラリ、および設定のみがコンテナ内にパッケージ化されます。
これにより、コンテナは仮想マシンと比較して、より軽量で、ポータブル（可搬性があり）、かつ効率的なデプロイモデルになります。

このコンテナ基盤の上に構築された link:https://openshift.com/[OpenShift,window='_blank'] は、ワークフローを合理化し、検証済みの統合を含む完全なプラットフォームを提供することで、効率的なアプリケーション開発を可能にします。これにより、プロジェクトを加速させたい開発チームや、標準化されたアプリケーションプラットフォームを探している組織にとって理想的なものとなっています。
